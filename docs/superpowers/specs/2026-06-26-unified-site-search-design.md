# Unified site search — design

**Date:** 2026-06-26
**Status:** Approved (design) — implementing
**Author:** drafted with Claude

## 1. Goal

Replace the three inconsistent search mechanisms (Pokémon ARIA-combobox + fuzzy,
Sets inline filter-the-list, Artist plain input with no feedback) with ONE
consistent search that shows suggestions as you type. Add a global search
(homepage + header ⌘K) that searches across all sections, grouping results by
type and hiding empty groups. Section pages get the same component scoped to
their type.

Best-practice basis: the command-palette pattern (cmdk — already a dependency;
used by Linear / Vercel / Algolia DocSearch): grouped, keyboard-first, hide
empty groups, type icons.

## 2. Decisions (resolved in brainstorming)

- **Searchable types:** Pokémon, Sets, Artists, FAQs, Facts. **Not** individual
  cards (20k+, reached via their set/Pokémon pages).
- **Placement:** homepage global box + header ⌘K (every page) + section-scoped on
  /pokemon, /illustrator, /sets.
- **/sets:** the scoped typeahead replaces the inline filter; the browse-by-era
  list stays.

## 3. Architecture

### 3.1 Compact index via a server route (fetched once, searched client-side)
`app/api/search-index/route.ts` returns a flat JSON array of
`{ type: 'pokemon'|'set'|'artist'|'faq'|'fact', label, sublabel?, url }`,
assembled server-side from:
- **Sets** — `getSetsForSitemap()` (cache-only, never blocks on the API; all
  warmed sets) → `{set, seoName, series · year, /set/<id>}`.
- **Pokémon** — entity catalog (`pokemonCatalog()`, 1,020) → `#dex`, `/pokemon/<slug>`.
- **Artists** — gated artist catalog (`gatedArtistEntries()`, 287) → cards count,
  `/illustrator/<slug>`.
- **FAQs** — `ALL_FAQ_PAGES` (registry) → question, `/faqs/<slug>`.
- **Facts** — `ARTICLES` → question, `/facts/<slug>`.

~1,800 entries, ~150KB (gzip ~40KB), `Cache-Control` long-lived. The client
**fetches it once on first search focus** (module-memoised promise) and searches
locally — instant after load, no per-keystroke network. Chosen over a committed
build artifact because FAQ pages are generated by runtime TS templates a `.mjs`
build can't enumerate, and a route keeps the index fresh with no regeneration.

### 3.2 Search engine (pure, TDD'd) — `lib/search/`
- `fuzzy.ts`: generalise `pokemon-search`'s accent-fold key + scoring. `score(query,
  label)` ranks exact > prefix > word-start > substring > subsequence/edit-distance;
  returns null for no match. Accent- and punctuation-insensitive.
- `search.ts`: `runSearch(query, entries, { scope?, perGroup=6 })` → ordered
  `SearchGroup[]` (`{ type, label, items }`), empty groups dropped, each capped.
  Group order: Pokémon, Sets, Artists, FAQs, Facts. Scope = one type → single group.
- `pokemon-search.ts` keeps its "did you mean" API but delegates scoring to the
  shared fuzzy core (no duplicate algorithm).

### 3.3 UI (cmdk, Game-Boy themed) — `components/search/`
- Restyle `components/ui/command.tsx` to the GB aesthetic (pixel-font group
  headers, 3px borders, gb palette) — it's currently unused shadcn default styling.
- A pixel icon per type (reuse `entity-icons`; add set/faq/fact glyphs).
- `SiteSearch` core: cmdk with `shouldFilter={false}`, fed the ranked/grouped
  results; cmdk handles keyboard nav + ARIA + selection. `scope` prop → global vs
  one type. Selecting navigates to the result `url` (info pages for entities).
- Two shells: `SiteSearchBox` (inline input + dropdown — homepage + sections) and
  `SiteSearchDialog` (`CommandDialog`, opened by a header button or ⌘K — global,
  every page).
- Index lazy-loaded via `useSearchIndex()` (fetch + memo) on first focus.

### 3.4 Placement & replacement
- **Header**: a "Search ⌘K" control → global dialog; ⌘K / Ctrl-K shortcut.
- **Homepage**: prominent global `SiteSearchBox` near the top.
- **/pokemon** → replace `PokemonTypeahead` (scope=pokemon).
- **/illustrator** → replace `IllustratorSearchForm` (scope=artist).
- **/sets** → replace the SetsBrowser inline filter (scope=set); keep the era list.
- Delete `pokemon-typeahead`, `illustrator-search-form`, and the sets filter UI.

## 4. Accessibility
cmdk provides combobox/listbox roles + keyboard; the dialog is a focus-trapped
modal with a labelled input. ⌘K shortcut with a visible hint. axe-clean.

## 5. Testing
- Unit (TDD): `fuzzy` (exact/prefix/typo/accent ordering), `search` grouping
  (caps, empty-hidden, order, scope), route assembly shape.
- Component render: `SiteSearchBox` global (groups render, empty hidden, select
  navigates), scoped (one type), dialog opens via ⌘K, axe.
- Browser verification on prod: type → grouped suggestions; ⌘K; section scoping;
  keyboard nav.

## 6. Out of scope
- Searching individual cards (reachable via set/Pokémon pages).
- Server-side per-keystroke search / external search service.
- Recent-searches / history (could follow later).
